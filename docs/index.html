연구자를 위한 베이지안 최적화 (Bayesian Optimization) 튜토리얼

목표: 새로운 반도체 소재의 '열처리 온도(X)'에 따른 '전기 전도도(Y)'의 최댓값을 가장 적은 실험으로 찾아라!

1. 노가다(?) 실험의 한계
2. 똑똑한 추론 (대리 모델)
3. 다음 실험은 어디로? (탐색 전략)
4. 실전 최적화 루프

1. 그리드 탐색 (Grid Search)의 한계

보통 최적의 온도를 찾을 때 100도, 200도, 300도... 이렇게 일정한 간격으로 모두 실험해보는 방식을 씁니다. 하지만 한 번 실험할 때마다 엄청난 비용과 시간이 든다면 어떨까요?
아래 버튼을 눌러 일정한 간격으로 실험을 진행해 보세요. 최고 성능을 찾기 위해 너무 많은 실험이 낭비됩니다.

현재 실험 횟수: 0번

2. 모르는 것을 아는 것 (대리 모델, Surrogate Model)

베이지안 추론에서는 몇 개의 초기 실험 데이터만으로 전체 그래프의 모양을 유추합니다 (파란 실선).
데이터가 없는 곳의 불확실성을 파란색 음영으로 계산합니다.
💡 드롭다운 메뉴에서 커널을 변경해 가며, 인공지능이 곡선을 '얼마나 매끄럽게' 이으려고 하는지 비교해 보세요!

3. 활용(Exploitation) vs 탐색(Exploration)

이제 다음 실험을 어디서 할지 결정해야 합니다. 두 가지 전략이 충돌합니다.
1. 활용(안전빵): 지금 예측했을 때 가장 성능이 좋을 것 같은 곳 (평균이 높은 곳)
2. 탐색(모험): 아직 한 번도 실험해보지 않아서 결과가 어떨지 전혀 모르는 곳 (불확실성이 높은 곳)

이 두 가지를 조율해 다음 실험 위치의 가치를 평가하는 함수를 획득 함수(Acquisition Function)라고 합니다. 현재 사용 중인 함수는 가장 대표적인 UCB (Upper Confidence Bound)이며, 수식은 다음과 같습니다.
UCB(x) = μ(x) + κ · σ(x)
여기서 μ(x)는 예측된 성능(평균), σ(x)는 예측된 불확실성(표준편차)이며, κ(카파)는 모험을 얼마나 감수할지 결정하는 가중치 파라미터입니다.

아래 슬라이더를 움직여 연구자의 성향(κ)을 바꿔보세요. 하단 녹색 그래프의 최고점(★)이 다음 실험 추천 위치입니다.
활용 중심 (안전) 탐색 중심 (모험)
현재 탐색 가중치 (κ): 2.0

4. 실전! 베이지안 최적화 루프

이제 AI에게 운전대를 맡겨봅시다. 베이지안 최적화 알고리즘이 '현재 지식 바탕으로 추론 → 획득 함수로 다음 실험 위치 추천 → 실험 수행 → 지식 업데이트' 과정을 스스로 반복합니다.
그리드 탐색(Step 1)과 비교해서 얼마나 빨리 최고점을 찾아내는지 확인해 보세요!

현재 실험 횟수: 2번 (초기 데이터 2개 포함)
